\doxysubsection{Detailed Description}
Wrapper for aws-\/c-\/io channel handlers. The semantics are identical as the functions on aws\+\_\+channel\+\_\+handler.

All virtual calls are made from the same thread (the channel\textquotesingle{}s thread).

The channel calls shutdown on all handlers twice, once to shut down reading, and once to shut down writing. Shutdown always begins with the left-\/most handler, and proceeds to the right with dir set to \mbox{\hyperlink{namespace_aws_1_1_crt_1_1_io_a26cb072017a059e42ff798325aff6e1fa7a1a5f3e79fdc91edf2f5ead9d66abb4}{Channel\+Direction\+::\+Read}}. Then shutdown is called on handlers from right to left with dir set to \mbox{\hyperlink{namespace_aws_1_1_crt_1_1_io_a26cb072017a059e42ff798325aff6e1fa1129c0e4d43f2d121652a7302712cff6}{Channel\+Direction\+::\+Write}}.

The shutdown process does not need to complete immediately and may rely on scheduled tasks. The handler MUST call \mbox{\hyperlink{class_aws_1_1_crt_1_1_io_1_1_channel_handler_ae05cfa451288067c74811c43530381f7}{On\+Shutdown\+Complete()}} when it is finished, which propagates shutdown to the next handler. If \textquotesingle{}free\+Scarce\+Resources\+Immediately\textquotesingle{} is true, then resources vulnerable to denial-\/of-\/service attacks (such as sockets and file handles) must be closed immediately before the shutdown process complete. \mbox{\Hypertarget{class_aws_1_1_crt_1_1_io_1_1_channel_handler_a3056d7241980af565b1009164169d2b0}\label{class_aws_1_1_crt_1_1_io_1_1_channel_handler_a3056d7241980af565b1009164169d2b0}}
